Cambio local de entorno para ejecutar sin Docker:
- Se configuro una base PostgreSQL local separada del servicio global del sistema.
- Motivo: las credenciales del servicio PostgreSQL existente no estaban disponibles/funcionales para automatizar `develop`.
- Archivo afectado: `.env` local.
- Valores usados:
  - `POSTGRES_HOST=127.0.0.1`
  - `POSTGRES_PORT=5434`
  - `POSTGRES_DB=app_db`
  - `POSTGRES_USER=app_user`
  - `POSTGRES_PASSWORD=app_password`

Impacto:
- Los scripts locales de arranque (`scripts/start-backend.ps1`) toman estos valores del `.env`.
- `backend/src/config/settings.py` ahora carga `.env` local con `python-dotenv` sin sobrescribir variables reales del entorno.
- Esto permite ejecutar comandos Django sin Docker y no deberia afectar Docker porque `docker-compose.yml` sigue inyectando sus variables.
- Si otro entorno no levanta, revisar primero si esta usando Docker, PostgreSQL del sistema o esta base local separada.
